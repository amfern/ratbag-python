probe(device)

Probe the device for information. On success, the driver will create a ratbag.Device with at least one profile and the appropriate number of buttons/leds/resolutions.

A caller should subscribe to the device-added signal before calling this function.

If device is a string or a pathlib.Path object, the driver creates initializes for the device at that path. Otherwise, device is used as-is as backing device instance. This is used when emulating devices - note that the exact requirements on the device behavior may differ between drivers.

Completion of this function without an exception counts as success.

Parameters

  • device – The path to the device or a fully initialized instance representing a device

  • device_info – Static information about the device from external sources (system, data files, etc).

  • config – Driver-specific device configuration (e.g. quirks).

Bases: ratbag.drivers.Driver

Implementation of the Logitech HID++ 2.0 protocol.

Driver options supported:

  • Buttons: the number of buttons exported by the device

  • DeviceIndex: the HID++ device index

  • Leds: the number of LEDs exported by device

  • ReportRate: fixed report rate

  • Quirk: the quirk to apply to this device, one of G305, G602

Parameters

  • config – A dict of the (lowercase) driver options

  • quirk – None or one of Quirk

class ratbag.drivers.Driver

Bases: gi.overrides.GObject.Object

The parent class for all driver implementations. See ratbag/drivers/drivername.py for the implementation of each driver itself.

A driver must implement the DRIVER_LOAD_FUNC() function, it is the entry point for ratbag to instantiate the driver for the device.

DRIVER_LOAD_FUNC = 'load_driver'

The name of the function to instantiate a driver. A driver file must contain at least the following code:

class MyDriver(ratbag.Driver):
    def probe(self, device, info, config):
        pass

def load_driver(driver_name=""):
    return myDriver()

GObject Signals:

  • device-added: emitted for each ratbag.Device that was added during probe()
